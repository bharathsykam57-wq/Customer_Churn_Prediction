# Customer Churn Prediction

A machine learning project to predict customer churn for a telecom company.
Built using a dataset of 7043 customers with 20 features.

## Project Structure

```
customer-churn-prediction/
    data/
        Customer-Churn.csv          Raw dataset (21 columns, 7043 rows)
        tel_churn.csv               Generated by EDA notebook (one-hot encoded)
    models/
        ada_boost_churn_model.pkl   Production model loaded by the Streamlit app
    notebooks/
        01_EDA.ipynb                Exploratory Data Analysis with detailed explanations
        02_ML_Model_Building.ipynb  Model training, comparison, and hyperparameter tuning
    app/
        app.py                      Streamlit web application
    requirements.txt
    .gitignore
    README.md
```

## Setup Instructions

### 1. Clone the repository

```bash
git clone <your-repo-url>
cd customer-churn-prediction
```

### 2. Create and activate a virtual environment

```bash
python3 -m venv .venv
source .venv/bin/activate
```

On Windows:

```bash
python -m venv .venv
.venv\Scripts\activate
```

### 3. Install dependencies

```bash
pip install -r requirements.txt
```

## Running the Notebooks

Start Jupyter from the project root:

```bash
jupyter notebook
```

Open the notebooks in order:

1. `notebooks/01_EDA.ipynb` - Run this first; it generates `data/tel_churn.csv`
2. `notebooks/02_ML_Model_Building.ipynb` - Trains all models and saves `.pkl` files to `models/`

## Running the Streamlit App

From the project root:

```bash
streamlit run app/app.py
```

The app will open in your browser at `http://localhost:8501`.

To run in the background (persistent):

```bash
nohup streamlit run app/app.py &
```

## Dataset

- Source: Telecom customer churn dataset
- Rows: 7043 customers
- Columns: 21 (20 features + 1 target)
- Target: `Churn` (Yes/No) - whether the customer left in the last month
- Class distribution: 73% No Churn, 27% Churn (imbalanced)

Key features:

- `tenure`: months the customer has been with the company
- `Contract`: month-to-month, one year, two year
- `MonthlyCharges`: monthly bill amount
- `TotalCharges`: total amount charged
- `InternetService`, `OnlineSecurity`, `TechSupport`: service subscriptions

## Models Trained

| Model | Imbalance Handling |
|---|---|
| Decision Tree (baseline) | None |
| Decision Tree | StandardScaler |
| Decision Tree | SMOTEENN |
| Random Forest (500 trees) | SMOTEENN |
| XGBoost | scale_pos_weight |
| XGBoost | SMOTEENN |
| XGBoost | SMOTE |
| XGBoost | ADASYN |
| AdaBoost | sample_weight (saved as production model) |
| Random Forest | class_weight=balanced |
| CatBoost | auto_class_weights=Balanced |
| LightGBM | scale_pos_weight |
| XGBoost | RandomizedSearchCV tuning |
| XGBoost | Optuna tuning |

The AdaBoost model with sample weighting is saved as `ada_boost_churn_model.pkl`
and is loaded by the Streamlit app.

## Key Insights from EDA

1. Churn rate is 26.5% - dataset is imbalanced (73:27)
2. Month-to-month contract customers churn at 41%
3. Electronic check payment method has the highest churn
4. Customers in their first 12 months are most likely to churn
5. Higher monthly charges correlate with higher churn
6. Customers with no online security or tech support churn more
7. Gender has negligible impact on churn
8. Long-term contract customers (2 year) have very low churn

## Evaluation Metric

Accuracy alone is misleading for imbalanced datasets.
The models are evaluated primarily using:

- F1 score (balance of precision and recall)
- Recall for class 1 (catching actual churners is the business priority)
- Classification report (full breakdown per class)
